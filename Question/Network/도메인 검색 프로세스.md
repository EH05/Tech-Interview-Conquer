# 도메인 검색 프로세스

---

## 면접 질문 예시

> "특정 도메인을 브라우저에 입력했을 때 일어나는 일을 순차적으로 설명해 주세요."

---

<details>
  <summary><h2> 평가 요소 및 평가 요소 별 모범 답안</h2></summary>

  ### 1. 도메인 이름 입력 및 요청
  - 브라우저는 사용자가 입력한 **URL을 분석**하여 프로토콜, 도메인, 경로 등을 파악한다.
  - 해당 도메인의 **IP 주소를 캐시에서 먼저 확인한 다음**에 캐시에 IP 주소가 있다면 DNS 조회없이 바로 해당 IP 주소로 요청을 보내고, 주소를 찾지 못하면 DNS 조회를 한다.


  ### 2. DNS 조회
  - 포함내용
    - DNS (Domain Name System): 도메인과 IP 주소를 매핑하는 계층적 시스템
    - 브라우저는 설정된 DNS 서버를 통해 도메인의 **IP 주소를 조회**한다. 이 과정에서 로컬 DNS 서버 (ISP) → 루트 네임서버 → TLD 네임서버 → 권한 있는 네임서버를 거쳐 IP 주소를 찾는다.


  ### 3. TCP 연결
  - 포함내용
    - 브라우저가 DNS를 통해 얻은 IP 주소로 서버와의 연결을 시도하기 위해 **TCP 3-way Handshake를 수행**한다.
    - TCP 3-way handshake : 클라이언트와 서버간의 신뢰성 있는 데이터 전송을 보장하기 위해 연결을 설정하는 과정. 클라이언트가 서버와 연결을 맺기 위해 SYN 패킷을 보내면, 서버가 클라이언트에게 응답으로 SYN-ACK 패킷을 보낸다. 클라이언트는 이후 ACK 패킷을 보내서 서버의 요청을 확인하고 연결을 확립한다.
    - HTTPS 환경에서는 TLS Handshake가 추가적으로 진행되며, 서버의 SSL 인증서를 확인한 후 암호화된 통신을 시작한다.

  ### 4. HTTP 요청 및 응답
  - 브라우저는 **HTTP 요청을 생성**하여 웹 서버로 보낸다.
  - 서버는 HTTP 요청을 받고 필요한 작업을 수행한 후 **HTTP 응답을 브라우저로 전송**한다.

  ### 5. 콘텐츠 렌더링
  - 브라우저는 HTML을 파싱하여 DOM (Document Object Model)을 생성하고, CSS를 파싱하여 CSSOM (CSS Object Model)을 만든다.
  - 이후 두 개를 결합하여 렌더 트리를 구성하고, 레이아웃 (배치 계산) → 페인트 (화면 출력) → 합성 (최적화) 단계를 거쳐 최종적으로 웹 페이지를 표시한다.
